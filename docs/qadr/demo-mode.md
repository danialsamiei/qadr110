# Demo Mode (دمو) — QADR110

هدف از «حالت دمو» این است که QADR110 حتی بدون کلیدهای بیرونی (OpenRouter/Groq/کانکتورهای اختیاری) قابل نمایش و تست UX باشد.

## فعال‌سازی

روش‌های فعال‌سازی (هرکدام کافی است):

- `VITE_DEMO_MODE=1` در فایل `.env.local`
- اضافه‌کردن `?demo=1` به URL (در اولین اجرا روی `localStorage` ذخیره می‌شود)
- تنظیم دستی `localStorage`:
  - کلید `qadr110-demo-mode` با مقدار `1`

## رفتار در حالت دمو

- لایه AI به‌جای تماس شبکه، **خروجی deterministic fixture** تولید می‌کند.
- بسته‌های دانش دمو به `knowledge packs` اضافه می‌شوند و با برچسب‌های `demo` و `synthetic` مشخص هستند.
- خروجی‌ها صریحاً هشدار می‌دهند که نمونه هستند و برای استفاده عملی باید کانکتورها/کلیدها فعال شوند.

## نقاط ورود UI

- پنل `کارگاه هوشمند QADR110`:
  - نشان `دمو`
  - دکمه `نمونه دمو` برای تولید یک گفت‌وگوی نمونه
- پنل `کارگاه ژئو-تحلیل`:
  - دکمه `بارگذاری نمونه دمو` (وقتی نتیجه‌ای وجود ندارد) برای اجرای چند تحلیل نقشه‌محور نمونه

## نکته‌ی ممیزی و شفافیت

دمو به‌صورت عمدی از داده‌های واقعی بیرونی استفاده نمی‌کند تا از «تلقین قطعیت» و ساختن روایت‌های نادقیق جلوگیری شود. هر خروجی دمو باید به‌عنوان نمونه UX/ساختار تحلیل خوانده شود، نه گزارش عملیاتی.
